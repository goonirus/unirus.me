The Overview 项目概览

Crazy Ludo is a modern, web-based implementation of the classic Chinese family game “Fei Xing Qi” (flying chess). Instead of trying to simulate the physical board 1:1, this version focuses on a clean layout and offline-first experience, built around a custom 52-grid track.

The entire project is written from scratch using Vanilla JavaScript and the HTML5 Canvas API, handling everything from board rendering to token movement and AI turns without relying on external engines or frameworks.

Crazy Ludo 是一款基于网页的现代版“飞行棋”实现。 相比一比一还原实体棋盘，本项目更关注清晰的布局与离线友好的体验， 围绕一条自定义的 52 格主环路来展开设计。

整个项目完全使用 原生 JavaScript 与 HTML5 Canvas API 搭建， 从棋盘绘制、棋子移动到 AI 回合逻辑全部自行实现， 不依赖任何外部游戏引擎或前端框架。

Key Features & Mechanics 核心特色与机制

Hybrid AI System 人机混合对战

Supports mixed gameplay with both Human and AI players. The AI automatically handles dice rolling and move selection, allowing for single-player practice or filling empty seats.

支持人类玩家与 AI 混合对局。AI 会自动完成掷骰与行动选择， 既可以用于单人练习，也可以在人数不足时补齐座位。

Custom Track Logic 自定义路线逻辑

Features a 52-grid outer loop with corner bases, including classic “Jump” (same color +4) and “Eat” (capture opponent) rules to keep interaction sharp and punishing.

使用 52 格外环与四角基地的自定义棋盘结构，完整实现 “同色前进 +4 格” 与 “吃子回家” 等经典飞行棋规则， 保证对抗性与双方互动的火花。

The game also includes a 6-grid Victory Runway that requires an exact roll to finish, creating natural tension in the late game. The launch pads are carefully aligned with the track entrance so that the visual flow from base to main ring feels smooth and readable.

游戏为每位玩家设计了 6 格终点冲刺跑道，必须掷出刚好的点数才能抵达终点， 使得后期节奏自然紧绷。起飞点与主环路入口在视觉和逻辑上保持对齐， 让棋子从基地飞入主路的过程更加顺畅、易于理解。

Technical Implementation 技术实现

A key challenge was drawing a responsive board that stays crisp at different resolutions while still matching the feel of a physical Ludo track. All geometry is generated procedurally based on canvas size, so the layout can be scaled without redrawing assets.

本项目的一大难点，是在不同分辨率下依然保持清晰的棋盘与格子布局， 同时保留类似实体飞行棋的观感。棋盘几何完全通过 Canvas 按尺寸动态计算生成， 因此可以在缩放画布时无损放大，不需要重新导出美术资源。

  • Canvas Rendering: Functions like buildRing() and drawBoard() construct the entire “hollow plus” shape and individual cells on the fly.
  • 棋盘绘制： 通过 buildRing()、drawBoard() 等函数， 动态绘制“中空十字”棋盘结构以及所有格子。
  • State Machine: A central S object tracks current player, dice result, token positions and phase flags, enabling features like auto-skipping when no piece can move.
  • 状态机设计： 使用中心状态对象 S 记录当前玩家、骰子结果、 棋子位置与阶段标记，实现“无子可走自动跳过”等质量提升功能。
  • Dynamic UI: The “Roll Dice” button is positioned relative to the canvas, so it stays accessible on both desktop and smaller laptop screens without covering important areas.
  • 动态 UI： “掷骰”按钮根据 Canvas 尺寸与位置动态布局， 在大屏与小屏上都能保持就手，又不会遮挡关键信息区域。
